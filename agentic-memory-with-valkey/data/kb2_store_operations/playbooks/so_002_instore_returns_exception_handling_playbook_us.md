# In-Store Returns Exception Handling Playbook (US)

## Purpose
Guide store associates and support agents on handling common in-store return edge cases while maintaining compliance with current returns policy and local store operations notes.

## Baseline Rules
This playbook does not replace returns policy. It explains how to operationalize approved policy in common exception scenarios.

## Common Scenarios

### 1) Customer has item but no packing slip
- Attempt return lookup using order number, payment method, or customer account per POS capabilities
- If order is found and item is eligible, proceed under normal return rules
- If order cannot be found, follow no-proof-of-purchase flow (if enabled in region/store)

### 2) Customer wants to return online order in store
- Confirm item category is eligible for in-store return
- Confirm order is not marketplace seller-fulfilled unless seller policy allows in-store return
- If eligible, process via omnichannel return flow in POS

### 3) Customer says item arrived damaged but packaging discarded
- Inspect item condition
- If defect is visible and policy permits, process per damaged-item return guidance
- For ambiguous condition cases, escalate to manager review using local workflow

### 4) Long line / reduced staffing periods
- Encourage digital queue or appointment process where available
- Do not deny eligible returns solely due to queue volume
- If system outage occurs, follow temporary outage advisory if active

## Escalation Triggers
Escalate to manager or lead if:
- Policy ambiguity
- Suspected fraud pattern
- Marketplace ownership unclear
- System error prevents return completion
- Customer dispute requires appeasement outside associate authority

## Override Precedence
1. Temporary policy exception
2. Active store/regional note
3. Returns policy
4. This playbook (operational guidance)
