# Store Pickup & Curbside Operations Playbook (US)

## Purpose
This playbook provides standard operating guidance for store pickup and curbside support interactions for U.S. stores. Use this document as the baseline process unless a regional/store-specific note or temporary advisory overrides it.

## Scope
Applies to:
- Buy Online Pickup In Store (BOPIS)
- Curbside pickup (where available)
- Contact-center and store associate handling guidance

## Baseline Pickup Readiness Workflow
1. Customer places order for eligible pickup items.
2. Store receives order and begins picking.
3. Order status changes to "Preparing for Pickup."
4. Once picked and staged, status changes to "Ready for Pickup."
5. Customer receives notification with pickup instructions.

## Standard Pickup Window
- Default pickup hold window: **5 calendar days** after "Ready for Pickup"
- On day 6, unclaimed orders may be canceled and refunded per order management rules
- Do not extend pickup hold windows unless local note or temporary exception exists

## Curbside Availability Baseline
Curbside is available only at stores enabled for curbside service and only during posted curbside hours. Some stores may temporarily suspend curbside due to weather, staffing, safety, or property restrictions.

## Customer Verification Requirements
Before handoff, confirm:
- Order confirmation or pickup barcode
- Government-issued ID (or approved alternate verification flow where configured)
- If pickup person differs from purchaser, verify designated pickup authorization in order record

## Common Exceptions and Handling

### Customer arrives before "Ready for Pickup"
- Apologize and confirm order status
- Do not promise a pickup time earlier than current estimate
- Offer notification confirmation and store callback only if local process supports it

### Customer says "I got the email but store cannot find item"
- Confirm order number and pickup name
- Ask store to re-check staging area and service desk hold shelf
- If not located within local search threshold, follow missing pickup escalation path

### Customer requests curbside at a store that does not offer curbside
- Explain store-specific service availability
- Offer in-store pickup if order is eligible
- If customer cannot enter store, review alternate pickup options if policy allows

## Override Precedence
When documents conflict, use this order:
1. Active temporary store advisory
2. Active regional/store-specific note
3. This playbook (baseline)

## Agent Response Guidance
Use clear, operational language:
- "I can check whether your store currently supports curbside pickup."
- "Your order must show Ready for Pickup before the store can release it."
- "Some stores have temporary service limits based on staffing or weather."
