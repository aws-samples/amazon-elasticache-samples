# Marketplace Seller Escalation Routing Playbook (US)

## Purpose
Standardize when and how support agents escalate marketplace seller issues to the Marketplace Seller Support team.

## Escalation Triggers
Escalate when any of the following apply:
1. Seller misses response SLA
2. Customer reports repeated unresolved contact attempts
3. Conflicting seller responses across the same claim
4. Defect/safety issue with potential customer risk
5. Repeated similar complaints on same SKU/seller in short period
6. Ownership ambiguity (seller vs retailer vs manufacturer warranty) blocks resolution

## Required Case Data Before Escalation
- Order ID
- Seller ID
- SKU / ASIN-equivalent identifier
- Return/claim reason
- Customer summary (1 to 2 sentences)
- Photos/evidence status
- Prior seller responses (if any)
- Urgency flag (standard / safety / high-value)

## Escalation Priorities
- **Urgent:** Safety/electrical overheating reports, potential injury risk
- **High:** Missed SLA + customer impacted near return window expiration
- **Standard:** Routine dispute or ownership clarification

## Agent Guidance
- Do not promise escalation outcome
- Do confirm expected next step and that case has been routed
- Use seller-specific docs plus partner master facts before escalating
