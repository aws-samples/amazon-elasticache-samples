# Case Pattern: Customer Wants to Use SwiftPace Watch with Two Phones

## Pattern Summary
Low-moderate frequency (avg 3 cases/week). Customer wants to pair SwiftPace with both a personal phone and a work phone, or switch between iPhone and Android.

## Current Limitation
SwiftPace can only be paired with **one phone at a time**. Pairing with a second phone requires unpairing from the first, which erases locally cached activity data that hasn't synced.

## Resolution
1. Explain the limitation clearly: "SwiftPace currently supports pairing with one phone at a time."
2. **If switching permanently** (e.g., got a new phone): Sync all data on old phone first (App → Sync Now). Then on new phone: install app, sign in with same account, pair watch. Watch will unpair from old phone automatically. All historical data is in the cloud account, not lost.
3. **If wanting to use two phones simultaneously:** Not supported. Workaround: use the Accent Athletics web dashboard (dashboard.accentathletics.com) on the second device to view data. Real-time sync only goes to the paired phone, but dashboard updates within 15 minutes.
4. **If switching between iPhone and Android frequently:** Each switch requires re-pairing. Watch faces and settings transfer via cloud account, but some phone-specific settings (notification preferences) must be reconfigured. This is tedious — be honest that it's not a great experience for frequent switchers.

## Feature Request
This is a top-10 feature request. Multi-device pairing is on the product roadmap but no ETA. If customer asks, say: "I've noted your interest in multi-device support. It's something our product team is actively working on, but I don't have a timeline to share."

## Stats
- First-contact resolution rate: 78% (lower because the answer is often "not supported")
- Avg handle time: 6 min
- Customer satisfaction: 3.4/5 (customers are disappointed by the limitation)
